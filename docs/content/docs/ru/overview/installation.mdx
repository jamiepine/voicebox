### macOS

<Tabs items={["Apple Silicon", "Intel"]}>
  <Tab value="Apple Silicon">
    Скачать: [voicebox_aarch64.app.tar.gz](https://github.com/jamiepine/voicebox/releases/latest/download/voicebox_aarch64.app.tar.gz)

    ```bash
    # Распаковать архив
    tar -xzf voicebox_aarch64.app.tar.gz

    # Переместить в Applications
    mv Voicebox.app /Applications/
    ```
  </Tab>
  <Tab value="Intel">
    Скачать: [voicebox_x64.app.tar.gz](https://github.com/jamiepine/voicebox/releases/latest/download/voicebox_x64.app.tar.gz)

## Первый запуск

При первом запуске Voicebox:

1. **Скачает модели** — первый TTS-движок, с которым вы начнёте генерировать,
   автоматически загрузит свои веса. Размеры варьируются от ~350 MB
   (Kokoro) до ~8 GB (TADA 3B). Для старта чаще всего выбирают Qwen 1.7B
   (~3.5 GB).
2. **Создаст директорию данных** — voice profiles и сгенерированное аудио
   хранятся в:
   - macOS: `~/Library/Application Support/sh.voicebox.app/`
   - Windows: `%APPDATA%/sh.voicebox.app/`
   - Linux: `~/.config/sh.voicebox.app/`
3. **Поднимет backend server** — встроенный Python-сервер стартует автоматически

<Callout type="info">
  Первая генерация обычно медленнее из-за скачивания и инициализации моделей.
  Все последующие запуски используют локальный кеш.
</Callout>

## Системные требования

### Минимальные

- **OS:** macOS 11+, Windows 10+ или Linux
- **RAM:** 8GB
- **Хранилище:** 5GB свободного места
- **CPU:** современный многоядерный процессор

### Рекомендуемые

- **RAM:** 16GB+
- **GPU:** CUDA-совместимая NVIDIA GPU для более быстрой генерации
- **Хранилище:** 10GB+ свободного места

<Callout type="info">
  CPU inference поддерживается, но работает заметно медленнее GPU. Для
  realtime-сценариев CUDA-совместимая видеокарта очень желательна.
</Callout>

## Проверка

После установки проверьте базовый сценарий:

1. Запустите Voicebox
2. Посмотрите на индикатор статуса сервера в левом нижнем углу — он должен
   быть зелёным
3. Перейдите во вкладку **Profiles** и создайте тестовый профиль
4. Сгенерируйте короткий аудиофрагмент, чтобы убедиться, что TTS-движок работает

<Callout type="success">
  Если индикатор статуса зелёный и вы можете сгенерировать аудио, всё готово.
</Callout>
