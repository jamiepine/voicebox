## Обзор

Remote Mode позволяет запускать бэкенд Voicebox на отдельной машине, например
на GPU-сервере, а само настольное приложение использовать локально у себя.

## Когда это особенно полезно

- **Нет локальной GPU** — можно использовать облачный GPU или удалённую рабочую станцию
- **Нужна более быстрая генерация** — тяжёлая обработка уходит на мощное железо
- **Общая инфраструктура** — к одному серверу могут подключаться несколько пользователей
- **Работа с ноутбука** — ноутбук остаётся холоднее и меньше тратит батарею

## Архитектура

В Remote Mode настольное приложение Voicebox, запущенное локально, общается с
удалённым бэкенд-сервером по HTTP. Локальный клиент отвечает только за
интерфейс, а удалённый сервер берёт на себя всю тяжёлую обработку: TTS-модели,
API-эндпоинты и генерацию аудио.

## Настройка Remote Mode

### На сервере

<Steps>
  <Step title="Установите зависимости">
    ```bash
    # Склонируйте репозиторий
    git clone https://github.com/jamiepine/voicebox.git
    cd voicebox/backend

    # Установите Python-зависимости
    pip install -r requirements.txt

    # Движки с несовместимыми транзитивными версиями — ставим через --no-deps
    pip install --no-deps chatterbox-tts
    pip install --no-deps hume-tada

    # Qwen3-TTS из исходников
    pip install git+https://github.com/QwenLM/Qwen3-TTS.git
    ```

    Либо просто выполните `just setup` из корня репозитория: этот сценарий
    автоматизирует всю подготовку.
  </Step>

  <Step title="Запустите сервер">
    ```bash
    # Разрешить внешние подключения
    uvicorn main:app --host 0.0.0.0 --port 17493
    ```

    <Callout type="warn">
      Так сервер становится доступен в сети. Для безопасности используйте
      firewall, VPN или другой ограничивающий слой.
    </Callout>
  </Step>

  <Step title="Откройте порт в firewall">
    ```bash
    # Ubuntu/Debian
    sudo ufw allow 17493

    # Либо настройте firewall у облачного провайдера
    ```
  </Step>
</Steps>

### На клиенте

<Steps>
  <Step title="Откройте настройки">
    В Voicebox перейдите в **Settings → Server**.
  </Step>

  <Step title="Включите Remote Mode">
    Активируйте переключатель **Use Remote Server**.
  </Step>

  <Step title="Укажите URL сервера">
    ```text
    http://<server-ip>:17493
    ```

    Замените `<server-ip>` на IP-адрес вашего сервера.
  </Step>

  <Step title="Проверьте соединение">
    Нажмите **Test Connection**, чтобы убедиться, что сервер доступен.
  </Step>
</Steps>

## Вопросы безопасности

<Callout type="warn">
  Сейчас у API нет встроенной аутентификации. Используйте Remote Mode только в
  доверенной сети или через VPN.
</Callout>

**Практические рекомендации:**

- использовать VPN вроде WireGuard или Tailscale вместо прямой публикации в интернет
- ставить обратный прокси с аутентификацией, например nginx + basic auth
- использовать HTTPS и SSL-сертификаты
- ограничивать доступ firewall-правилами по IP

## Производительность

Ожидаемая скорость на разных GPU:

| GPU | Скорость генерации |
|---|---|
| RTX 4090 | около 2-3 секунд на 10 слов |
| RTX 3090 | около 3-4 секунд на 10 слов |
| RTX 3060 | около 5-7 секунд на 10 слов |
| CPU (12 ядер) | около 20-30 секунд на 10 слов |

<Callout type="info">
  Для комфортной работы желательно иметь GPU как минимум с 8 ГБ VRAM.
</Callout>
